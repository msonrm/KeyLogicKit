## iOS 版との実装の違い・Android 固有機能

1. BT 経由のイベント順序不整合やアナログ値ジッターに対応するため、D-pad / LT / RT の状態を KeyEvent 由来と MotionEvent 軸由来の別フィールドで追跡。

2. LT/RT はヒステリシス（press 0.5 / release 0.15）+ 「っ」後 300ms の「ん」発火ブロックで BT 特有のチャタリングを吸収。LS クリックは 250ms の falling-edge debounce で二重確定（commit/OSC 送信が 2 回走る現象）を防ぐ（DualSense 等の機械式スティックボタン対策、当初 120ms で導入したが実機で 2 度押し化が頻発するため拡張）。

3. 「D-pad 同士 / フェイスボタン同士は排他」という前提を置いて chord 判定をシンプル化、within-family の変化は別文字として扱う。

4. 日本語変換エンジンは **KazumaProject/JapaneseKeyboard (MIT)** の converter を `com/kazumaproject/` に vendor して利用（LOUDS trie + N-gram 言語モデル、assets ~15MB）。学習・ユーザー辞書は Phase A4 で Room 実装済み: `com/gime/android/learn/GimeDatabase.kt` に単一 DB、`repository/LearnRepository.kt` + `UserDictionaryRepository.kt` で DAO 委譲。変換確定時（`GamepadInputManager.commitConversion()`）に文節単位と全体読みの両方を書き込み、スコア減衰（3000→500）で優遇。予測変換（`getCandidates` API）は未導入だが、`getCandidatesWithoutPrediction` でも `GraphBuilder` が `LearnRepository` を参照するため学習効果はラティスに反映される。

5. 変換は文節単位編集方式: 左スティック ←→ で文節フォーカス移動、↑↓ で focused 文節の候補 cycle（ページ単位ウィンドウ 9 件表示）。LS=確定、RS=バッファ全削除。「変換中:」行は文節ごとの**現在選択中 surface**（例 `私は|考えている`）を並べてプレビュー表示し、フォーカスを動かさずに全文節のレコメンドを一覧できるようにしてある。

6. 韓国語 받침 は iOS 版の「即時適用+巻き戻し」方式で 겹받침・LB単押し・LB+D-pad・子音先行/母音先行の全パターンに対応。**자모 모드（子音・母音単体入力、Android 先行実装）**: ㅋㅋㅋ 等のチャット表現用。LT 長押しで Jamo Lock（持続）、LT 2連続短押しで Smart Jamo（空白/句読点/削除/カーソル移動/LS/モード切替で自動解除）。有効時は合成をバイパスし D-pad/LB は互換 Jamo 子音（U+3131..U+314E）、フェイスボタンは母音（U+314F..U+3163）を直接 emit。右スティック → で直前 jamo を連打（ㅋㅋㅋ）、↑ で 평→격→경 サイクル（ㄲㄸㅃㅆㅉ/ㅋㅌㅍㅊ アクセス）。자모 모드中の LT 単独短押しは ㅇ 互換 jamo を emit（「ㅇㅇ」のため。row 0 は chord で到達不可なので LT に割当て）。モード突入時は composing を確定。通常モードの LT 単独短押しは従来通り ㅇ받침（press edge から release edge 発火に変更、長押し/2連続との衝突回避）。2連続短押しの1回目は ㅇ받침が発火する副作用あり（許容）。ビジュアライザ LT ラベル: 通常=「ㅇ」/ Smart=「자모」/ Lock=「LOCK」。

7. ビジュアライザは iOS 版と同じ統一レイアウト構造（言語切替で形が変わらない、ボタンフェイスだけ差し替える設計）。上段 = `[LT][LB]` / `[RB][RT]` 横並び、中段 = `[LS◯] [D-pad 5cell] [Face 4dir] [RS◯]`、下段は廃止（情報は全てスティック内に統合）。D-pad セル = 角丸矩形 8dp、フェイスボタン (X/Y/A/B) = 円、LS/RS スティック = 直径 60dp の円内に 3×3 グリッドを敷いて方向別の役割ラベルを直接表示（カーソル: 細い `↑↓←→` / 特殊アクション: 太い `⇧⇩⇦⇨` / Devanagari: varga 代表子音 क/च/ट/त）。`tightTextStyle()` ヘルパーで Compose Text のフォント余白を削除して全要素を視覚的中央に揃える。Material You 全面対応（colorScheme トークンで dynamic color、ダーク/ライト追従）。実装は `ui/GimeApp.kt` の `DpadDisplay` / `DpadCluster3x3` / `FaceButtons3x3` / `StickIndicator`。

8. **IME 化** (Phase A6): `GimeInputMethodService` が Service 自身に LifecycleOwner / ViewModelStoreOwner / SavedStateRegistryOwner を実装し、ViewTree owner を `window.decorView` に設定することで `ComposeView` ホスティングを実現。`InputMethodService.onGenericMotionEvent` を直接 override すれば joystick も focus 奪取なしで届く。`setComposingText` で composing 下線、LS で確定。詳細: `docs/gime-android-ime-plan.md`。**IME 折りたたみ + 候補フローティング化** (Phase A8+1): IME View を `Column [透過オーバーレイ(候補/composing) | compact バー(固定高 ~36dp) | (展開時) D-pad]` 構造に再構成。`onComputeInsets` で `contentTopInsets = compact バー top`（Compose の `positionInRoot()` で測定）とし、`TOUCHABLE_INSETS_CONTENT` でオーバーレイ上のタップはアプリへフォールスルー。compact バーの高さは言語切替を除いて固定なので、候補の表示/非表示でアプリの入力欄の縦位置は変わらない。候補表示ロジックは `GamepadVisualizer` から `CandidateOverlay` として切り出し、IME とメイン画面の両方で再利用。compact/展開状態は `ImeUiSettings` (SharedPreferences、`bubble_ui` とは別ファイル) で永続化、タイトルバーの ▾/▴ でトグル。

9. **VRChat OSC** (Phase A7): `com/gime/android/osc/` に外部依存なしの自前 OSC 1.0 実装。設定で有効化すると IME 入力を InputConnection と並行して `/chatbox/input` `/chatbox/typing` に送信（dual output）。LS 累積モデル（複数文節を 1 メッセージで送信、空状態で LS = 確定送信）、debounce 100ms、`MAX_CHATBOX_LEN = 144` でトリム。デフォルト OFF、ユーザーが明示的に有効化するまでソケット open しない。`docs/gime-vrchat-osc.md` がセットアップガイド。

10. **設定画面** (`ui/SettingsScreen.kt`): 言語モードの有効化トグル + 切替順序の上下入替 + ライセンス表示（KazumaProject/JapaneseKeyboard MIT, CC-CEDICT CC-BY-SA 4.0, libchewing LGPLv2.1 ほか）。モード設定は `GimeModeSettings` (SharedPreferences) で永続化し、MainActivity/IME 両方の InputManager が起動時にロードする。ユーザーが現在モードを OFF にした場合は `updateEnabledModes` が先頭モードへ自動遷移（バッファ類もクリア）。

11. **バブル表示** (Phase A8, `com/gime/android/bubble/`): `BubbleService` (foreground, specialUse) + `BubbleView` でフローティングオーバーレイ。`SYSTEM_ALERT_WINDOW` + `TYPE_APPLICATION_OVERLAY` 経由で `WindowManager.addView` + `ComposeView` ホスティング（Service 自身が LifecycleOwner 系を実装し view ルートに setViewTree*Owner）。`FLAG_NOT_FOCUSABLE` を `dispatchTouchEvent` の ACTION_DOWN / ACTION_OUTSIDE で動的に付け外しし、「タップで active（α=1.0・gamepad 受信）」「外タップで inactive（α=0.45・下の VRChat へ fallthrough）」を両立。サービスが独自の `GamepadInputManager` + `VrChatOscOutput` を保持し、IME と独立運用（IME を OFF にしてもバブル単独で chatbox 送信可能）。`observeComposingState` で日本語「変換せず LS」の buffer→empty 遷移を監視して composingText を accumulated に移す（IME 側と同じ作法）。`onDirectInsert` 非 composing 分岐で `replaceCount` を尊重して韓国語 받침 / T9 rollback の「旧字残り」を回避。**OSC 運用トグル** (`VrChatOscSettings`): `commitOnlyMode`（/chatbox/input 下書き抑制、VRChat Mobile で chatbox UI が開くのを回避）/ `typingIndicatorEnabled`（/chatbox/typing 送信の独立トグル）/ `autoReleaseAfterSend`（LS 送信後にバブルを自動非アクティブ化しフォーカスを下へ返す）/ `customTypingEnabled`（composing 開始/終了エッジで任意の avatar parameter に `int` / `float` / `bool` を送る。"考え中ポーズ" 等に活用、VRCEmote=7 プリセット同梱、IME・バブル両方で有効）。**UI 設定** (`BubbleSettings`): `compactMode` で D-pad を折りたたみ、プレビュー + 候補のみの省スペース表示に切替（タイトルバー ▾/▸）。タイトルバーは「GIME」+ 言語モードチップ + ▾/▸ + ×。OSC enable 時は ✈ プレビュー行で「これから送る下書き」を表示。

12. **Devanagari モード** (Phase A9, `engine/DevanagariComposer.kt` + `GamepadResolver.kt` の Devanagari テーブル群): `GamepadInputMode.DEVANAGARI` として 6 番目のモードを追加。Sanskrit / Hindi / Marathi / Nepali 等を gamepad 直接打鍵。**varnamala 時計回り** 方式（क→च→ट→त→प, क→ख→ग→घ, a→i→u→e を時計回りに配置）。**LS 前置シフトラッチ**: LS を direction に flick すると latch、中立に戻しても保持、左手側出力（D-pad 子音 / LB 鼻音）発火で latch 自動消費＝NEUTRAL 復帰、同方向再 flick で toggle off、別方向で上書き（左親指で LS と D-pad を同時操作できない物理制約への対応。毎回 `LS方向flick → D-pad押下` の同じリズムでブラインド向き）。**L3 one-shot 非 varga サブレイヤー**: L3 で enter、非 varga 子音 (य र ल व / श ष स ह) を 1 文字 emit すると自動 OFF。**合成モデル**: ITRANS / Google Hindi IME と同じく conjunct は halant (RT) を明示的に打つ（自動 conjunct はしない。`नम` を `न्म` にしないため）。**修飾子**: RT tap=halant ्、RT+LS 方向=カーソル移動、RT+LS click=改行（`devaRtUsedForCursor=true` で RT release 時の halant 自動挿入を抑止）、LT+RT=visarga ः、RB 単押し=ओ、LT+RB=nukta ़、LT+A=ऋ、RS ↑ anusvara ं↔chandrabindu ँ、RS → 長母音 post-shift (a→ā, e→ai, o→au 等)、RS ↓ space/।/॥ サイクル、RS ← backspace。詳細設計: `docs/gime-brahmic-expansion-memo.md`。

13. カメラモード（iPad カメラ + Vision Hand Pose の構想）は iOS 版でも削除済み、両プラットフォーム未実装（SPEC.md に先行アイデアとして記載）。

14. **VRChat OSC 二段送信翻訳** (Phase A10, `com/gime/android/translate/`): LS commit で日本語を `/chatbox/input` に送信した直後、裏で **ML Kit On-Device Translation** に投げて、結果を **notification=false** で同じ chatbox に上書き送信する。VRChat 側では「日本語送信音 → 静かに翻訳が表示」の挙動になり、翻訳レイテンシが UX 演出に変わる。完全オフライン処理。**ターゲット言語**: `OFF / English / 한국어 / 中文(简体) / 中文(繁體・台湾)` の 5 種。`VrChatOscSettings.translationTarget` に永続化。**モデル DL**: 言語ペアごとに ~30MB を on-demand DL、デフォルト WiFi 限定。設定画面に「事前 DL + テスト」ボタン。**上書きまでの最低遅延** (`translationOverwriteDelayMs`): 0 / 1s / 1.5s / 2s / 3s から選択（デフォルト 1.5s）。翻訳と並行カウントなので、翻訳完了がそれより早ければ残り時間を待つ。原文を読む間が確保され、デモ動画キャプチャの視認性も上がる。**繁體（台湾）後処理** (`ChineseConverter`): ML Kit は中文を簡体しか持たないため、OpenCC4j (`com.github.houbb:opencc4j:1.14.0`, Apache 2.0, 純 JVM, JNI 不要) の `DataMaps.taiwan().stPhrase()` から 49,000 件の S→T フレーズマップを取り出し、自前の greedy longest-match で適用してから `ZhConverterUtil.toTraditional()` で残り単漢字を S→T 変換する 2 段構成。`軟件→軟體` `視頻→影片` `網絡→網路` `信息→資訊` `互聯網→網際網路` `打開→開啟` 等の台湾語彙置換が効く（`ZhConvertBootstrap.toTraditional()` 経由では台湾辞書 lookup が走らないバグを回避）。**配線**: IME (`GimeInputMethodService`) / バブル (`BubbleService`) 両方の `onConfirmOrNewline` で `commit()` 直後に `scheduleTranslationFollowup()` を呼ぶ。iOS (GiME) への移植は今後対応予定。詳細: `docs/gime-vrchat-osc.md` の「二段送信翻訳」セクション。

15. **LT+LS click = Ctrl+Enter 送信** (お試し実装、`GamepadInputManager.onCtrlEnter` callback): Slack / Discord / X / ChatGPT 等の「Ctrl+Enter で送信」UI を gamepad だけで踏めるようにする。LS click handler で `gp.ltValue > triggerPressThreshold` を検出して分岐。composing 中（日/中/韓）はまず確定してから `onCtrlEnter` を発火（「打鍵 → LT 押したまま LS」で 1 chord 確定 + 送信）。LT release 側の副作用（拗音後置 / 「っ」/ Smart Caps / ㅇ받침）は `ltConsumedByCtrlEnter` フラグで抑止し、各モードの LT release handler と handleSnapshot の LT press edge の両方で reset（Devanagari/Chinese など LT release を持たないモード経由の stale 抑止対策）。Devanagari の `RT+LS click = 改行` と衝突しないよう、`LT && RT` 同時押し時は通常経路に戻す。送出先は (a) `GimeInputMethodService`: `InputConnection.sendKeyEvent` で `META_CTRL_ON | META_CTRL_LEFT_ON` 付き `KEYCODE_ENTER` の DOWN/UP（VRChat OSC 有効時は通常 commit に倒す）、(b) `BubbleService`: host への InputConnection が無いので chord 無反応を避けて OSC commit と同じパスに倒す、(c) `GimeApp` (MainActivity): ローカル TextField に検証用 `⌃↵` マーカーを挿入。iOS (GiME) への移植は今後対応予定。
