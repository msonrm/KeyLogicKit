# AR Box Keyboard — 空箱オーバーレイキーボード構想

*Author: Masao Narumi ([@msonrm](https://github.com/msonrm))*
*Date: April 7, 2026*
*License: MIT*

## 概要

手に持った空箱の表面に、AR でキーボードレイアウトを投影する仮想キーボードの構想。Apple Vision Pro 等のハンドトラッキング対応 HMD を想定する。

物理キーボードを持ち歩かずに、触覚フィードバック付きの安定したタイピング体験を実現する。

## 背景: 既存 AR/VR キーボードの課題

| 方式 | 課題 |
|------|------|
| 空中タイピング（Meta Quest 等） | 接触面がなく指が空振りする。疲れやすく精度が低い |
| パススルーで物理キーボードを使用 | 物理キーボードが必要。AR の意味が薄い |
| 机の表面をタップ | 手の位置がずれるとキー配置もずれる。座標が不安定 |

いずれも「座標の安定性」「触覚フィードバック」「携帯性」のうち少なくとも1つを欠いている。

## 提案: 空箱ホールド方式

四角い空箱（iPhone、リモコン、小さな箱など）を横向きに両手で持ち、箱の表面にキーボードレイアウトを AR オーバーレイで投影する。

```
┌─────────────────────────────┐
│  Q  W  E  R  T  Y  U  I  O  P  │  ← AR オーバーレイ
│   A  S  D  F  G  H  J  K  L    │    （箱の表面に投影）
│    Z  X  C  V  B  N  M         │
└─────────────────────────────┘
         ↑ 空箱（iPhone 等）
     左手で保持 ←──→ 右手で保持
```

### なぜ空箱を持つのか

1. **座標安定性**: 箱が両手と一体になって動くため、手と仮想キーの相対位置が常に一定。机タイピングのような座標ずれが発生しない
2. **物理的な接触面**: 指が箱の表面に触れる瞬間を打鍵として検出できる。空中タイピングの「空振り」問題を解消
3. **触覚フィードバック**: iPhone を箱として使えば、Core Haptics で打鍵ごとに振動パターンを返せる。物理キーボードの打鍵感に近い体験が得られる
4. **携帯性**: キーボードを持ち歩く必要がない。手持ちの iPhone や適当な箱で成立する
5. **配列の自由**: 物理キーの制約がないため、QWERTY に限らず任意のキーボードレイアウトを投影可能

### 技術的な実現方法

#### ハンドトラッキング

- **Apple Vision Pro**: ARKit Hand Tracking API で両手の関節点を常時取得。OS レベルで高精度なトラッキングが提供される
- **iPad（フォールバック）**: Vision Framework の Hand Pose Detection でカメラ映像から関節点を取得

#### 打鍵検出

1. ARKit で箱の表面（平面）を認識
2. 各指先（thumbTip, indexTip 等）の3D 座標と箱の表面との距離を計算
3. 指先が表面に接触（距離がしきい値以下）した瞬間を打鍵イベントとして発火
4. 接触位置をキーボードレイアウト上の座標に変換し、対応するキーを特定

#### 触覚フィードバック（iPhone を箱として使用する場合）

1. Vision Pro と iPhone を BLE / Multipeer Connectivity で接続
2. 打鍵検出イベントを iPhone に送信
3. iPhone が Core Haptics で振動パターンを再生
4. イベント種別（通常打鍵・修飾キー・変換確定等）に応じて異なる触覚パターン

#### 入力パイプライン

```
ARKit Hand Tracking
  → 指先 - 箱表面の接触検出
  → 接触座標 → キーマップ上のキー特定
  → KeyEvent（HIDKeyCode + modifiers）
  → KeyRouter → KeyAction → InputManager（かな漢字変換）
```

GiME / KanaEditor の既存パイプライン（KeyRouter → InputManager）をそのまま利用できる。キーマップ定義（JSON）を差し替えるだけで、ローマ字・AZIK・月配列・NICOLA 等あらゆる配列で日本語入力が可能。

## 先行研究との差別化

本構想の核心は「空箱を持つ」という行為が、AR キーボードの3つの根本的課題を同時に解決する点にある:

| 課題 | 空箱ホールドによる解決 |
|------|----------------------|
| 座標安定性 | 箱が手と一体化し、相対位置が常に固定 |
| 触覚フィードバック | 箱の物理的な接触面 + iPhone の Core Haptics |
| 携帯性 | 手持ちの iPhone や任意の箱で成立 |

加えて、物理キーの制約がないことで:

- **任意のキーボードレイアウトを動的に投影**できる（言語・配列を瞬時に切替）
- **キーサイズや配置を手の大きさに合わせて調整**できる
- **同時打鍵系配列**（NICOLA、月配列等）の親指キー位置を最適化できる

## 応用

- **多言語入力**: 言語切替でキーボードレイアウトを瞬時に変更（日本語 → 英語 → 韓国語 → 中国語）
- **アクセシビリティ**: 物理キーボードが使えない環境での代替入力手段
- **モバイルワーク**: カフェ、電車、立ち作業など、キーボードを置く平面がない環境でのテキスト入力
