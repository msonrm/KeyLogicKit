# ゲームパッド入力 マッピング仕様書

ゲームパッド（W3C Standard Gamepad Layout 準拠）で日本語・英語・韓国語・中国語簡体字・中国語繁體字を入力する方式。
Start ボタンでモードを切り替える。

## 設計原則

1. **左手=子音/行選択、右手=母音/文字選択**: ゲームパッドの左右分離を活用
2. **同時押しで1文字入力**: D-pad（左手）とフェイスボタン（右手）の chord
3. **Eager output + rollback**: ボタン押下の瞬間に即出力し、300ms 以内の変更で差し替え
4. **後置シフト**: 入力後に修飾（拗音、濁点、子音サイクル等）。同時押し数を削減
5. **モード共通操作**: LS 押込み=確定/改行、RS 押込み=キャンセル、右スティック←=BS

## 入力モード

| モード | バッジ色 | 初期 | パイプライン |
|--------|---------|------|-------------|
| 日本語 | 桜色(pink) | ✓ | GCController → GamepadInputManager → **InputManager**（IME 経由） |
| 韓国語 | indigo | | GCController → GamepadInputManager → **onDirectInsert**（IME バイパス） |
| 英語 | 緑 | | GCController → GamepadInputManager → **onDirectInsert**（IME バイパス） |
| 中国語簡体 | 赤 | | GCController → GamepadInputManager → **PinyinEngine** → **onDirectInsert**（IME バイパス） |
| 中国語繁體 | 青 | | GCController → GamepadInputManager → **注音テーブル** → **PinyinEngine** → **onDirectInsert**（IME バイパス） |

サイクル順（Start ボタン）: デフォルトは **日本語 → 韓国語 → 英語 → 中国語簡体 → 中国語繁體 → 日本語**。設定シートで有効/無効の切替と、ドラッグによる順序変更が可能。

## 共通操作

| ボタン | 動作 |
|--------|------|
| LS 押し込み | 確定（composing 中、文節単位の部分確定）/ 改行（idle 時） |
| RS 押し込み | キャンセル（composing 破棄） |
| Back / Select | idle 時: テキスト操作モードトグル。composing 時: スペース |
| Start | モード切替 |
| Start + Back 同時押し | テキスト共有（共有シート表示） |
| 右スティック← | バックスペース（1文字削除） |

### 左アナログスティック

| 状態 | ← | → | ↑ | ↓ |
|------|---|---|---|---|
| idle（全モード） | カーソル左 | カーソル右 | — | — |
| composing（日本語） | 文節縮小 | 文節拡張 | 前候補 | 変換開始 |
| selecting（日本語） | 文節縮小 | 文節拡張 | 前候補 | 次候補 |

英語・韓国語モードでは常にカーソル移動。

### テキスト操作モード

Back ボタン（idle 時）でトグル。文単位のナビゲーション・選択・並べ替えを行う。
モード中は通常の文字入力・句読点入力を受け付けない。

| 操作 | アクション |
|------|-----------|
| Back ボタン | テキスト操作モード解除 |
| 左スティック ←/↑ | 前の文頭へカーソル移動 |
| 左スティック →/↓ | 次の文末へカーソル移動 |
| RB + 左スティック ←→ | 1文字ずつカーソル移動 |
| RB + 左スティック ↑↓ | 1行ずつカーソル移動 |
| RT + 左スティック ←/↑ | カーソル位置の文（または選択範囲）を前の文と入れ替え |
| RT + 左スティック →/↓ | カーソル位置の文（または選択範囲）を次の文と入れ替え |
| D-pad → | スマート選択 拡大（括弧内→括弧含む→文。文レベルが上限） |
| D-pad ← | スマート選択 縮小 |
| D-pad ↑ | 文単位選択: 後方に伸ばす / 前方選択中は1文縮める |
| D-pad ↓ | 文単位選択: 前方に伸ばす / 後方選択中は1文縮める |

| LB | T6 | 6 | m | n | o | |
| LB+← | T7 | 7 | p | q | r | s |
| LB+↑ | T8 | 8 | t | u | v | |
| LB+→ | T9 | 9 | w | x | y | z |
| LB+↓ | T0 | 0 | @ | # | - | _ |

| 操作 | 動作 |
|------|------|
| RT 単押し | 数字「0」入力 |
| 右スティック→ | スラッシュ (/) |
| 右スティック↑ | アポストロフィ |

## 韓国語モード

### 設計思想

2ボル式（두벌식）ベース。母音ボタン配置は字形の方向に対応（天地人の方向原理）。

### 子音（D-pad + LB）

D-pad で子音（초성）を選択。ㄱㄴㄷㄹ順（가나다라순）。

| 操作 | 子音 | 操作 | 子音 |
|------|------|------|------|
| なし | ㅇ | LB | ㅁ |
| ← | ㄱ | LB+← | ㅂ |
| ↑ | ㄴ | LB+↑ | ㅅ |
| → | ㄷ | LB+→ | ㅈ |
| ↓ | ㄹ | LB+↓ | ㅎ |

### 母音（フェイスボタン）— 字形方向対応

| ボタン | 位置 | 基本母音 | RT シフト（y系） |
|--------|------|---------|----------------|
| RB | 中立 | ㅡ（水平線） | ㅣ（垂直線） |
| X | 左 | ㅓ（左向き） | ㅕ |
| Y | 上 | ㅗ（上向き） | ㅛ |
| B | 右 | ㅏ（右向き） | ㅑ |
| A | 下 | ㅜ（下向き） | ㅠ |

### 音節入力

子音（D-pad）+ 母音（フェイスボタン）の同時押しで1音節を合成。

Unicode 合成式: `syllable = 0xAC00 + (onset × 21 + nucleus) × 28 + coda`

### 받침（終声）— 2ボル式スタイル

音節入力後、**全ボタンリリースを経てから子音単独入力**（母音ボタンなし）で받침を即時追加する。
LB+D-pad チョード（例: LB→LB+↑ でㅅ）では、LB 到着時に暫定パッチムを即適用し、
D-pad 追加で row が変わると巻き戻して再適用する（即時適用+巻き戻し方式）。
ボタンリリース開始で巻き戻しを停止し、全リリースでパッチムを確定する。
母音が後から来た場合はパッチムを巻き戻して新音節として処理する。
chord（子音+母音同時押し）で初声+中声が一括入力されるため、
キーボード2ボル式の「連音（받침の初声移動）」は不要。

| 操作 | 効果 |
|------|------|
| D-pad 単独（母音なし） | その子音の받침を追加 |
| LT 単独 | ㅇ받침を追加 |

**判定ルール**:
- D-pad + 母音（chord）= 常に新音節。前の音節は確定
- D-pad 単独 / LT 単独 = 常に받침。曖昧性なし

### 겹받침（二重終声）

받침がある状態でさらに子音単独入力:
- 겹받침テーブルに有効な組合せあり → 겹받침に合成
- なし → 現音節を確定

有効な겹받침（11組合せ）: ㄳ, ㄵ, ㄶ, ㄺ, ㄻ, ㄼ, ㄽ, ㄾ, ㄿ, ㅀ, ㅄ

### 複合母音・子音修飾

| 操作 | 効果 | 例 |
|------|------|-----|
| RT + フェイスボタン | y系母音 | ㅏ→ㅑ, ㅓ→ㅕ, ㅗ→ㅛ, ㅜ→ㅠ |
| RT 単押しリリース | ㅣ付加 | ㅏ→ㅐ, ㅓ→ㅔ, ㅗ→ㅚ, ㅘ→ㅙ, ㅜ→ㅟ, ㅝ→ㅞ, ㅡ→ㅢ |
| 右スティック↑ | 子音サイクル（받침があれば받침を、なければ초성をサイクル） | ㄱ→ㅋ→ㄲ→ㄱ, ㅅ→ㅆ→ㅅ |
| 右スティック→ | 複合母音（ㅏ/ㅓ付加） | ㅗ→ㅘ, ㅚ→ㅙ, ㅜ→ㅝ, ㅟ→ㅞ |
| 右スティック↓ 1回 | スペース | |
| 右スティック↓ 2回（400ms以内） | スペース→ピリオドに差し替え | |

### 入力例: 「한국어」

1. **한**: ㅎ(LB+↓) + ㅏ(B) → 하, ↑ 単独(ㄴ받침) → 한
2. **국**: ㄱ(←) + ㅜ(A) → 구, ← 単独(ㄱ받침) → 국
3. **어**: ㅇ(なし) + ㅓ(X) → 어

### 入力例: 「세종대왕」

1. **세**: ㅅ(LB+↑) + ㅓ(X) → 서, RT 単押し(ㅣ付加) → 세
2. **종**: ㅈ(LB+→) + ㅗ(Y) → 조, LT(ㅇ받침) → 종
3. **대**: ㄷ(→) + ㅏ(B) → 다, RT 単押し(ㅣ付加) → 대
4. **왕**: ㅇ(なし) + ㅗ(Y) → 오, R🕹→(ㅘ) → 와, LT(ㅇ받침) → 왕

### 入力例: 겹받침「많다」

1. **많**: ㅁ(LB) + ㅏ(B) → 마, ↑ 単独(ㄴ받침) → 만, LB+↓ 単独(ㅎ) → 겹받침 ㄶ → 많
2. **다**: ㄷ(→) + ㅏ(B) → 다

## 中国語共通

### テーブル構造

簡体字・繁体字ともに英語 T9 テーブルと同じ 5 列構造: `[RB(数字), X(左), Y(上), B(右), A(下)]`。
RB で行の数字（1〜0）を入力し、X/Y/B/A で文字（アルファベット or 注音記号）を入力する。

### 数字入力（簡体・繁体共通）

| ボタン | バッファ空 | バッファに文字あり |
|--------|----------|-----------------|
| RB | 行の数字（1〜0）挿入 | 無視 |
| RT 単押し | 「0」挿入 | 無視 |

### 候補操作（簡体・繁体共通）

| 操作 | アクション |
|------|-----------|
| 左スティック ↓ | 次の候補を選択 |
| 左スティック ↑ | 前の候補を選択 |
| LS 押込み | 選択中の候補を確定・挿入 |
| RS 押込み | バッファ・候補をクリア |
| 右スティック ← | バッファ末尾1文字削除（空ならバックスペース） |
| 右スティック → | 顿号「、」（バッファあれば先頭候補を暗黙確定） |
| 右スティック ↓ 1回 | 逗号「，」 |
| 右スティック ↓ 2回（400ms以内） | 逗号→句号「。」に差し替え |
| 右スティック ↓ 3回（400ms以内） | 句号→空白に差し替え |

### ビジュアライザ（簡体・繁体共通）

- D-pad: 英語モードと同じ十字配置（各ボタン内に X/Y/B/A 対応の文字を上下左右に表示）
- ピンインバッファ: バッジ横に表示（繁体字は注音記号で表示）

## 中国語簡体モード（简拼 = Abbreviated Pinyin）

英語 T9 テーブルを再利用してアルファベットを入力し、abbreviated pinyin（ピンインの頭文字）で候補を検索する。IME をバイパスし `onDirectInsert` で直接テキスト挿入。

辞書は CC-CEDICT + OpenSubtitles 頻度リスト上位 50,000 語から生成（~803KB、4,336 keys、16,017 候補）。各キーあたり最大 30 候補を頻度順に収録。

### 文字入力

英語モードと同じ T9 テーブルで小文字アルファベットを入力（フェイスボタン X/Y/B/A）。入力文字はピンインバッファに追加され、`PinyinEngine.lookup()` で候補を検索。

候補は最大9件ずつ表示するスライディングウィンドウ方式で、左スティック↑↓で全候補（最大30件）をスクロール可能。

### 入力例

1. **你好**: D-pad↓ + LB + X = `n`, D-pad→ + Y = `h` → 候補「你好」→ LS 確定
2. **知道**: D-pad↑ + X = `z`, D-pad↑ + X = `d`（LB解除後） → 候補「知道」→ LS 確定

## 中国語繁體モード（注音首 = Abbreviated Zhuyin）

注音符号テーブルから声母を入力し、abbreviated zhuyin で候補を検索する。台湾語彙（libchewing ベース）に最適化。

辞書は libchewing tsi.csv 上位 50,000 語から生成（~1.3MB、7,699 keys、23,452 候補）。各キーあたり最大 30 候補を頻度順に収録。候補表示は簡体字と同じスライディングウィンドウ方式。

### 注音テーブル（D-pad + LB で行選択、X/Y/B/A で注音選択、RB で数字）

ㄦ は abbreviated zhuyin で ㄜ と同じ "e" にマップされるため省略。

| 行 | 操作 | RB | X(左) | Y(上) | B(右) | A(下) |
|----|------|----|-------|-------|-------|-------|
| 唇音 | ニュートラル | **1** | ㄅ | ㄆ | ㄇ | ㄈ |
| 舌尖音 | ← | 2 | ㄉ | ㄊ | ㄋ | ㄌ |
| 舌根音 | ↑ | 3 | ㄍ | ㄎ | ㄏ | — |
| 舌面音 | → | 4 | ㄐ | ㄑ | ㄒ | — |
| そり舌音 | ↓ | 5 | ㄓ | ㄔ | ㄕ | ㄖ |
| 舌歯音 | LB | 6 | ㄗ | ㄘ | ㄙ | — |
| 単母音 | LB+← | 7 | ㄚ | ㄛ | ㄜ | ㄝ |
| 複母音 | LB+↑ | 8 | ㄞ | ㄟ | ㄠ | ㄡ |
| 鼻母音 | LB+→ | 9 | ㄢ | ㄣ | ㄤ | ㄥ |
| 介母 | LB+↓ | 0 | ㄧ | ㄨ | ㄩ | — |

### 入力例

1. **學校**: ㄒ(→+B) + ㄒ(→+B) → 候補「學校」→ LS 確定
2. **軟體**: ㄖ(↓+A) + ㄊ(←+Y) → 候補「軟體」→ LS 確定

## 入力タイミング

### Eager Output + Rollback

母音ボタン押下の瞬間に即出力し、同時打鍵ウィンドウ（300ms）内に子音が変化した場合は差し替え。全3モード共通のパターン。

```
Frame 0: RB 押下 → 即出力
Frame 3: D-pad← 押下 (50ms後) → 差し替え（300ms以内）
Frame 5: LB 押下 (83ms後) → 再差し替え（300ms以内）
...ボタンリリース → 確定
```

子音ボタンのリリース（ボタン数の減少）では巻き戻しを発火しない。

### 内部パラメータ

| パラメータ | 値 | 用途 |
|-----------|-----|------|
| chordWindow | 300ms | 同時打鍵判定ウィンドウ |
| doubleTapWindow | 400ms | 2度押し判定（句読点、Caps Lock 等） |
| stickThreshold | 0.5 | スティック入力判定閾値 |
| longPressThreshold | 500ms | LT 長押し判定（永続 Caps Lock） |

## ビジュアライザ

### モード別表示

- モードバッジ: ビジュアライザ最上部に常時表示（桜色/indigo/緑/赤/青）
- D-pad: 日本語/韓国語=ラベル表示、英語/簡体字/繁体字=十字配置（X/Y/B/A 対応）
- フェイスボタン: 日本語=かな、英語=文字（シフト状態反映）、韓国語=母音、簡体字=アルファベット、繁体字=注音記号
- RB: 日本語=あ段、英語/簡体字/繁体字=数字、韓国語=ㅡ
- ピンインバッファ: 中国語モード時、バッジ横に表示（繁体字は注音記号で表示）
- LT/LB 配置: 物理コントローラー準拠（LT=外側、LB=内側）
- RB/RT 配置: RB=内側、RT=外側

### 英語シフト状態

フェイスボタン文字と LT ラベルがシフト状態に連動:
- 通常: 小文字、LT=「SHIFT」
- シフト: 大文字（1文字のみ）、LT=「SHIFT」
- スマート Caps: 大文字、LT=「Caps」

## 実装ファイル

### ネイティブ版（Swift、GIME アプリ）

| ファイル | 役割 |
|----------|------|
| `GamepadResolver.swift` | かなテーブル、英語 T9 テーブル、モード enum、アクション enum |
| `GamepadInputManager.swift` | GCController 接続監視、スナップショット処理、5モード入力ハンドラ |
| `KoreanComposer.swift` | ハングル音節合成エンジン、子音/母音テーブル、サイクル/複合母音マップ |
| `PinyinEngine.swift` | CJK 候補検索エンジン（簡体: CC-CEDICT、繁体: libchewing） |
| `GamepadVisualizerView.swift` | SwiftUI ビジュアライザ（モード別表示、シフト状態反映） |
| `App.swift` | アプリエントリポイント、IMETextView 接続、コールバック配線 |

### Web 版（TypeScript）

| ファイル | 役割 |
|----------|------|
| `web/src/engine/gamepad-kana-table.ts` | かなテーブル、拗音マップ、濁点マップ |
| `web/src/hooks/useGamepadInput.ts` | rAF ポーリング、エッジ検出、eager output + rollback |
| `web/src/components/GamepadVisualizer.tsx` | ビジュアライザ（日本語モードのみ） |

### アーキテクチャ

```
キーボード:    pressesBegan → KeyEvent → KeyRouter → InputManager
ゲームパッド:  GCController → GamepadSnapshot → GamepadInputManager
                → InputManager（日本語: IME 経由）
                → onDirectInsert（英語/韓国語/中国語: IME バイパス）
```
