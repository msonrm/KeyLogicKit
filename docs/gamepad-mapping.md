# ゲームパッド入力 マッピング仕様書

ゲームパッド（W3C Standard Gamepad Layout 準拠）で日本語・英語・韓国語・中国語簡体字・中国語繁體字・Devanagari を入力する方式。
Start ボタンでモードを切り替える。

> **Devanagari モード**（GIME iOS / Android, Phase A9）は朗唱順ベースの独自 layout を採用しており、本ドキュメントのモード共通原則から外れる。詳細は [`docs/gime-brahmic-expansion-memo.md`](gime-brahmic-expansion-memo.md) を参照。

## 設計原則

1. **左手=子音/行選択、右手=母音/文字選択**: ゲームパッドの左右分離を活用
2. **同時押しで1文字入力**: D-pad（左手）とフェイスボタン（右手）の chord
3. **Eager output + rollback**: ボタン押下の瞬間に即出力し、300ms 以内の変更で差し替え
4. **後置シフト**: 入力後に修飾（拗音、濁点、子音サイクル等）。同時押し数を削減
5. **モード共通操作**: LS 押込み=確定/改行、RS 押込み=キャンセル、右スティック←=BS

| モード | バッジ色 | 初期 | パイプライン |
|--------|---------|------|-------------|

| 韓国語 | indigo | | GCController → GamepadInputManager → **onDirectInsert**（IME バイパス） |

## 共通操作

| ボタン | 動作 |
|--------|------|
| LS 押し込み | 確定（composing 中、文節単位の部分確定）/ 改行（idle 時） |
| RS 押し込み | キャンセル（composing 破棄） |
| Back / Select | idle 時: テキスト操作モードトグル。composing 時: スペース |
| Start | モード切替 |
| Start + Back 同時押し | テキスト共有（共有シート表示） |
| 右スティック← | バックスペース（1文字削除） |

### 左アナログスティック

| 状態 | ← | → | ↑ | ↓ |
|------|---|---|---|---|
| idle（全モード） | カーソル左 | カーソル右 | — | — |
| composing（日本語） | 文節縮小 | 文節拡張 | 前候補 | 変換開始 |
| selecting（日本語） | 文節縮小 | 文節拡張 | 前候補 | 次候補 |

英語・韓国語モードでは常にカーソル移動。

### テキスト操作モード

Back ボタン（idle 時）でトグル。文単位のナビゲーション・選択・並べ替えを行う。
モード中は通常の文字入力・句読点入力を受け付けない。

| 操作 | アクション |
|------|-----------|
| Back ボタン | テキスト操作モード解除 |
| 左スティック ←/↑ | 前の文頭へカーソル移動 |
| 左スティック →/↓ | 次の文末へカーソル移動 |
| RB + 左スティック ←→ | 1文字ずつカーソル移動 |
| RB + 左スティック ↑↓ | 1行ずつカーソル移動 |
| RT + 左スティック ←/↑ | カーソル位置の文（または選択範囲）を前の文と入れ替え |
| RT + 左スティック →/↓ | カーソル位置の文（または選択範囲）を次の文と入れ替え |
| D-pad → | スマート選択 拡大（括弧内→括弧含む→文。文レベルが上限） |
| D-pad ← | スマート選択 縮小 |
| D-pad ↑ | 文単位選択: 後方に伸ばす / 前方選択中は1文縮める |
| D-pad ↓ | 文単位選択: 前方に伸ばす / 後方選択中は1文縮める |

## 韓国語モード

### 設計思想

2ボル式（두벌식）ベース。母音ボタン配置は字形の方向に対応（天地人の方向原理）。

### 子音（D-pad + LB）

D-pad で子音（초성）を選択。ㄱㄴㄷㄹ順（가나다라순）。

| 操作 | 子音 | 操作 | 子音 |
|------|------|------|------|
| なし | ㅇ | LB | ㅁ |
| ← | ㄱ | LB+← | ㅂ |
| ↑ | ㄴ | LB+↑ | ㅅ |
| → | ㄷ | LB+→ | ㅈ |
| ↓ | ㄹ | LB+↓ | ㅎ |

### 母音（フェイスボタン）— 字形方向対応

| ボタン | 位置 | 基本母音 | RT シフト（y系） |
|--------|------|---------|----------------|
| RB | 中立 | ㅡ（水平線） | ㅣ（垂直線） |
| X | 左 | ㅓ（左向き） | ㅕ |
| Y | 上 | ㅗ（上向き） | ㅛ |
| B | 右 | ㅏ（右向き） | ㅑ |
| A | 下 | ㅜ（下向き） | ㅠ |

### 音節入力

子音（D-pad）+ 母音（フェイスボタン）の同時押しで1音節を合成。

Unicode 合成式: `syllable = 0xAC00 + (onset × 21 + nucleus) × 28 + coda`

### 받침（終声）— 2ボル式スタイル

音節入力後、**全ボタンリリースを経てから子音単独入力**（母音ボタンなし）で받침を即時追加する。
LB+D-pad チョード（例: LB→LB+↑ でㅅ）では、LB 到着時に暫定パッチムを即適用し、
D-pad 追加で row が変わると巻き戻して再適用する（即時適用+巻き戻し方式）。
ボタンリリース開始で巻き戻しを停止し、全リリースでパッチムを確定する。
母音が後から来た場合はパッチムを巻き戻して新音節として処理する。
chord（子音+母音同時押し）で初声+中声が一括入力されるため、
キーボード2ボル式の「連音（받침の初声移動）」は不要。

| 操作 | 効果 |
|------|------|
| D-pad 単独（母音なし） | その子音の받침を追加 |
| LT 単押し（短押し・release edge） | ㅇ받침を追加 |

**判定ルール**:
- D-pad + 母音（chord）= 常に新音節。前の音節は確定
- D-pad 単独 / LT 単独 = 常に받침。曖昧性なし

> **LT の発火タイミング (Android のみ)**: 長押し / 2連続短押しの判定と衝突しないよう、ㅇ받침 は press edge ではなく **release edge** で発火する。実体感は数十 ms 遅延するだけで支障はない。iPad 版は press edge 即発火のまま。

### 겹받침（二重終声）

받침がある状態でさらに子音単独入力:
- 겹받침テーブルに有効な組合せあり → 겹받침に合成
- なし → 現音節を確定

有効な겹받침（11組合せ）: ㄳ, ㄵ, ㄶ, ㄺ, ㄻ, ㄼ, ㄽ, ㄾ, ㄿ, ㅀ, ㅄ

### 複合母音・子音修飾

| 操作 | 効果 | 例 |
|------|------|-----|
| RT + フェイスボタン | y系母音 | ㅏ→ㅑ, ㅓ→ㅕ, ㅗ→ㅛ, ㅜ→ㅠ |
| RT 単押しリリース | ㅣ付加 | ㅏ→ㅐ, ㅓ→ㅔ, ㅗ→ㅚ, ㅘ→ㅙ, ㅜ→ㅟ, ㅝ→ㅞ, ㅡ→ㅢ |
| 右スティック↑ | 子音サイクル（받침があれば받침を、なければ초성をサイクル） | ㄱ→ㅋ→ㄲ→ㄱ, ㅅ→ㅆ→ㅅ |
| 右スティック→ | 複合母音（ㅏ/ㅓ付加） | ㅗ→ㅘ, ㅚ→ㅙ, ㅜ→ㅝ, ㅟ→ㅞ |
| 右スティック↓ 1回 | スペース | |
| 右スティック↓ 2回（400ms以内） | スペース→ピリオドに差し替え | |

### 자모 모드（子音・母音単体入力モード、Android 先行実装）

ㅋㅋㅋ, ㅎㅎㅎ, ㅠㅠㅠ などのチャット表現を直接入力するためのモード。
合成（ハングル音節組立て）をバイパスし、互換 Jamo（U+3131..U+3163）を 1 字ずつ出力する。
英語 Caps Lock と対称な 3 段階トグル:

| ジェスチャー | 効果 | 解除条件 | ビジュアライザ LT ラベル |
|-------------|------|---------|----------------------|
| LT 単独短押し（通常時） | ㅇ받침を追加（既存） | — | 「ㅇ」 |
| LT 2連続短押し（400ms以内） | Smart Jamo（一時モード） | 空白 / 句読点 / 削除 / カーソル移動 / LS 押込み / Start で自動解除 | 「자모」 |
| LT 長押し（500ms以上） | Jamo Lock（持続モード） | 再度 LT 長押し or Start で OFF | 「LOCK」 |

> **副作用**: 2連続短押しの 1 回目は ㅇ받침 が発火する（合成中・coda 無しのとき）。Smart Jamo を「合成のない idle 状態で起動するのが自然」というケースなら無害。逃げたい場合は長押しを使う。

#### 자모 모드中の入力

| 操作 | 効果 |
|------|------|
| D-pad 方向キー（単独 or LB+方向） | 互換 Jamo 子音を直接 emit（ㄱ ㄴ ㄷ ㄹ ㅁ ㅂ ㅅ ㅈ ㅎ） |
| LB 単独押し → LB 単独離し | release edge で ㅁ を emit（LB は修飾子扱い） |
| フェイスボタン単独 | 互換 Jamo 母音を emit（基本層 ㅡ ㅓ ㅗ ㅏ ㅜ） |
| RT + フェイスボタン | シフト層母音 ㅣ ㅕ ㅛ ㅑ ㅠ |
| LT 単押し（자모 モード中） | ㅇ 互換 jamo を emit（row 0 は chord で到達不可なため LT に割当） |
| 右スティック → | 直前 emit した jamo を 1 字追加（ㅋ → ㅋㅋ → ㅋㅋㅋ） |
| 右スティック ↑ | 直前子音を 평→격→경 サイクル（ㄱ→ㅋ→ㄲ）。母音には作用しない |
| 右スティック ↓ / ← | 既存通り（句読点/スペース、削除）。Smart Jamo はここで自動解除 |

#### LB の扱い

자모 モードでは LB は「シフト修飾子」として扱う:
- LB 押下エッジでは emit せず、D-pad 方向キーの後着を待つ
- D-pad 方向キーが絡んだ瞬間に row（LB 込み）を emit
- LB を D-pad 無しで離すと、release edge で ㅁ を emit
- ㅂ を出した後に LB だけ離した場合、ㄱ への refine はしない（直前選択を尊重）

#### モード突入時の挙動

突入時に合成中の音節を確定（commit）する。例: 「가」を composing 中に LT 長押し → 「가」確定 → Jamo Lock ON。

#### 入力例

- **ㅋㅋㅋ**: D-pad ← → 右スティック ↑ → 右スティック → → 右スティック → （ ㄱ → ㅋ → ㅋㅋ → ㅋㅋㅋ ）
- **ㅠㅠㅠ**: A + RT → 右スティック → → 右スティック → （ ㅠ → ㅠㅠ → ㅠㅠㅠ ）
- **ㅎㅎ**: LB+↓ → 離す → LB+↓ → 離す （ ㅎ → ㅎㅎ ）
- **ㅇㅇ**: LT 単押し → LT 単押し （ ㅇ → ㅇㅇ ）

### 入力例: 「한국어」

1. **한**: ㅎ(LB+↓) + ㅏ(B) → 하, ↑ 単独(ㄴ받침) → 한
2. **국**: ㄱ(←) + ㅜ(A) → 구, ← 単独(ㄱ받침) → 국
3. **어**: ㅇ(なし) + ㅓ(X) → 어

### 入力例: 「세종대왕」

1. **세**: ㅅ(LB+↑) + ㅓ(X) → 서, RT 単押し(ㅣ付加) → 세
2. **종**: ㅈ(LB+→) + ㅗ(Y) → 조, LT(ㅇ받침) → 종
3. **대**: ㄷ(→) + ㅏ(B) → 다, RT 単押し(ㅣ付加) → 대
4. **왕**: ㅇ(なし) + ㅗ(Y) → 오, R🕹→(ㅘ) → 와, LT(ㅇ받침) → 왕

### 入力例: 겹받침「많다」

1. **많**: ㅁ(LB) + ㅏ(B) → 마, ↑ 単独(ㄴ받침) → 만, LB+↓ 単独(ㅎ) → 겹받침 ㄶ → 많
2. **다**: ㄷ(→) + ㅏ(B) → 다

### 자모 모드（子音・母音単体入力モード）

「ㅋㅋㅋ」「ㅠㅠㅠ」のようなチャット表現や단독 자모を入力するための副モード。
LT を Caps Lock 様のジェスチャーで切替える（英語の Shift / Caps と対称設計）。

| LT ジェスチャー | 効果 |
|------|------|
| 長押し（≥500ms） | **Jamo Lock** トグル（持続モード、もう一度長押しで OFF） |
| 2連続短押し（≤400ms） | **Smart Jamo** トグル（一時モード、自動解除あり） |
| 単押し | 通常モード=ㅇ받침 / 자모 모드中=ㅇ jamo 単体入力 |

**Smart Jamo は次のアクションで自動解除されます**: 空白・句読点・削除・カーソル移動・改行（LS）・モード切替（Start）。Lock はこれらでは解除されません。

자모 모드中の入力:

| 操作 | 効果 |
|------|------|
| D-pad 方向キー単独 | 子音単体（互換 Jamo: ㄱ ㄴ ㄷ ㄹ など）。eager + chord window で LB 後着補正 |
| LB 単独 | release edge で ㅁ を emit（修飾子扱い、D-pad と組合せると emit しない） |
| LB + D-pad | LB 込み row で子音単体（ㅂ ㅅ ㅈ ㅎ など） |
| フェイスボタン単独 | 母音単体（基本層）。RT 同時で y系シフト層 |
| 右スティック → | 直前 jamo を 1 字追加（연타: ㅋㅋㅋㅋ） |
| 右スティック ↑ | 直前子音を 평→격→경 サイクル |
| LT 単押し | ㅇ 互換 jamo（D-pad/LB の chord 経路で到達不可なので LT に割当て） |

**ビジュアライザの LT ラベル**: 通常=「ㅇ」 / Smart Jamo=「자모」 / Lock=「LOCK」

**右スティックヒント**: 자모 モード中は「ㄱㅋㄲ / 연타」、通常モードは「ㅋㅌ / ㅘㅝ」と動的に切替。

> **既存挙動の変更**: 通常モードの LT 単押し ㅇ받침 は **press edge → release edge** 発火に変更（長押し / 2連続との衝突回避）。LT 2連続短押しの 1 回目では ㅇ받침 / ㅇ jamo が発火する副作用あり（許容された設計）。

## 中国語共通

### テーブル構造

簡体字・繁体字ともに英語 T9 テーブルと同じ 5 列構造: `[RB(数字), X(左), Y(上), B(右), A(下)]`。
RB で行の数字（1〜0）を入力し、X/Y/B/A で文字（アルファベット or 注音記号）を入力する。

### 数字入力（簡体・繁体共通）

| ボタン | バッファ空 | バッファに文字あり |
|--------|----------|-----------------|
| RB | 行の数字（1〜0）挿入 | 無視 |
| RT 単押し | 「0」挿入 | 無視 |

### 候補操作（簡体・繁体共通）

| 操作 | アクション |
|------|-----------|
| 左スティック ↓ | 次の候補を選択 |
| 左スティック ↑ | 前の候補を選択 |
| LS 押込み | 選択中の候補を確定・挿入 |
| RS 押込み | バッファ・候補をクリア |
| 右スティック ← | バッファ末尾1文字削除（空ならバックスペース） |
| 右スティック → | 顿号「、」（バッファあれば先頭候補を暗黙確定） |
| 右スティック ↓ 1回 | 逗号「，」 |
| 右スティック ↓ 2回（400ms以内） | 逗号→句号「。」に差し替え |
| 右スティック ↓ 3回（400ms以内） | 句号→空白に差し替え |

### ビジュアライザ（簡体・繁体共通）

- D-pad: 英語モードと同じ十字配置（各ボタン内に X/Y/B/A 対応の文字を上下左右に表示）
- ピンインバッファ: バッジ横に表示（繁体字は注音記号で表示）

## 中国語簡体モード（简拼 = Abbreviated Pinyin）

英語 T9 テーブルを再利用してアルファベットを入力し、abbreviated pinyin（ピンインの頭文字）で候補を検索する。IME をバイパスし `onDirectInsert` で直接テキスト挿入。

辞書は CC-CEDICT + OpenSubtitles 頻度リスト上位 50,000 語から生成（~803KB、4,336 keys、16,017 候補）。各キーあたり最大 30 候補を頻度順に収録。

### 文字入力

英語モードと同じ T9 テーブルで小文字アルファベットを入力（フェイスボタン X/Y/B/A）。入力文字はピンインバッファに追加され、`PinyinEngine.lookup()` で候補を検索。

候補は最大9件ずつ表示するスライディングウィンドウ方式で、左スティック↑↓で全候補（最大30件）をスクロール可能。

### 入力例

1. **你好**: D-pad↓ + LB + X = `n`, D-pad→ + Y = `h` → 候補「你好」→ LS 確定
2. **知道**: D-pad↑ + X = `z`, D-pad↑ + X = `d`（LB解除後） → 候補「知道」→ LS 確定

## 中国語繁體モード（注音首 = Abbreviated Zhuyin）

注音符号テーブルから声母を入力し、abbreviated zhuyin で候補を検索する。台湾語彙（libchewing ベース）に最適化。

辞書は libchewing tsi.csv 上位 50,000 語から生成（~1.3MB、7,699 keys、23,452 候補）。各キーあたり最大 30 候補を頻度順に収録。候補表示は簡体字と同じスライディングウィンドウ方式。

### 注音テーブル（D-pad + LB で行選択、X/Y/B/A で注音選択、RB で数字）

ㄦ は abbreviated zhuyin で ㄜ と同じ "e" にマップされるため省略。

| 行 | 操作 | RB | X(左) | Y(上) | B(右) | A(下) |
|----|------|----|-------|-------|-------|-------|
| 唇音 | ニュートラル | **1** | ㄅ | ㄆ | ㄇ | ㄈ |
| 舌尖音 | ← | 2 | ㄉ | ㄊ | ㄋ | ㄌ |
| 舌根音 | ↑ | 3 | ㄍ | ㄎ | ㄏ | — |
| 舌面音 | → | 4 | ㄐ | ㄑ | ㄒ | — |
| そり舌音 | ↓ | 5 | ㄓ | ㄔ | ㄕ | ㄖ |
| 舌歯音 | LB | 6 | ㄗ | ㄘ | ㄙ | — |
| 単母音 | LB+← | 7 | ㄚ | ㄛ | ㄜ | ㄝ |
| 複母音 | LB+↑ | 8 | ㄞ | ㄟ | ㄠ | ㄡ |
| 鼻母音 | LB+→ | 9 | ㄢ | ㄣ | ㄤ | ㄥ |
| 介母 | LB+↓ | 0 | ㄧ | ㄨ | ㄩ | — |

### 入力例

1. **學校**: ㄒ(→+B) + ㄒ(→+B) → 候補「學校」→ LS 確定
2. **軟體**: ㄖ(↓+A) + ㄊ(←+Y) → 候補「軟體」→ LS 確定

## 入力タイミング

### Eager Output + Rollback

母音ボタン押下の瞬間に即出力し、同時打鍵ウィンドウ（300ms）内に子音が変化した場合は差し替え。全3モード共通のパターン。

```
Frame 0: RB 押下 → 即出力
Frame 3: D-pad← 押下 (50ms後) → 差し替え（300ms以内）
Frame 5: LB 押下 (83ms後) → 再差し替え（300ms以内）
...ボタンリリース → 確定
```

子音ボタンのリリース（ボタン数の減少）では巻き戻しを発火しない。

### 内部パラメータ

| パラメータ | 値 | 用途 |
|-----------|-----|------|
| chordWindow | 300ms | 同時打鍵判定ウィンドウ |
| doubleTapWindow | 400ms | 2度押し判定（句読点、Caps Lock、Smart Jamo 等） |
| stickThreshold | 0.5 | スティック入力判定閾値 |
| longPressThreshold | 500ms | LT 長押し判定（永続 Caps Lock / Jamo Lock） |
| triggerPressThreshold / ReleaseThreshold | 0.5 / 0.15 | LT/RT のヒステリシス閾値（BT のジッター対策、Android のみ） |
| nBlockAfterTsu | 300ms | LT+RT=っ 出力後の「ん」発火抑止期間（Android のみ、BT 特有のチャタリング対策） |

## ビジュアライザ

### モード別表示

- モードバッジ: ビジュアライザ最上部に常時表示（桜色/indigo/緑/赤/青）
- D-pad: 日本語/韓国語=ラベル表示、英語/簡体字/繁体字=十字配置（X/Y/B/A 対応）
- フェイスボタン: 日本語=かな、英語=文字（シフト状態反映）、韓国語=母音、簡体字=アルファベット、繁体字=注音記号
- RB: 日本語=あ段、英語/簡体字/繁体字=数字、韓国語=ㅡ
- ピンインバッファ: 中国語モード時、バッジ横に表示（繁体字は注音記号で表示）
- LT/LB 配置: 物理コントローラー準拠（LT=外側、LB=内側）
- RB/RT 配置: RB=内側、RT=外側
- **VRChat OSC バッジ** (任意): VRChat OSC 連携が有効なときビジュアライザ左上に紫カプセルで「✈️ VRChat OSC」を表示。Activity 上ではタップで設定画面遷移、IME 上では表示のみ

### 英語シフト状態

フェイスボタン文字と LT ラベルがシフト状態に連動:
- 通常: 小文字、LT=「shift」
- シフト: 大文字（1文字のみ）、LT=「Shift」
- スマート Caps: 大文字、LT=「Caps」
- 永続 Caps: 大文字、LT=「CAPS」

### 韓国語 자모 モード状態（Android）

LT ラベルが자모 モードの状態に連動:
- 通常: LT=「ㅇ」（単押しで ㅇ받침 / 자모中なら ㅇ jamo）
- Smart Jamo（一時）: LT=「자모」
- Jamo Lock（持続）: LT=「LOCK」

자모 モード中は右スティックヒントも切替: ↑=「ㄱㅋㄲ」（cycle）、→=「연타」（repeat）。

#### Android 固有の差分

1. **トリガーヒステリシス + 「ん」抑止**: BT 経由の LT/RT アナログ値ジッター対策。`triggerPressThreshold` (0.5) と `triggerReleaseThreshold` (0.15) で press/release を分離。LT+RT=っ 出力後 300ms は「ん」発火を抑止
2. **D-pad / LT / RT は KeyEvent 由来と MotionEvent 軸由来を別フィールドで追跡**: BT のイベント順序不整合対策
3. **자모 모드（先行実装）**: iPad 版にはまだない。LT 長押し / 2連続タップで toggle
4. **設定永続化**: 言語モード有効化リストを SharedPreferences で保存（iPad 版は in-memory のみ）
5. **学習・ユーザー辞書**: Room で実装（KazumaProject `LearnRepository` / `UserDictionaryRepository` を vendor）
6. **テキスト操作モード・カメラモードは未移植**

### アーキテクチャ

```
キーボード:    pressesBegan → KeyEvent → KeyRouter → InputManager
ゲームパッド:  GCController → GamepadSnapshot → GamepadInputManager
                → InputManager（日本語: IME 経由）
                → onDirectInsert（英語/韓国語/中国語: IME バイパス）
```
