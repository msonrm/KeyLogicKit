# ゲームパッドひらがな入力 マッピング仕様書

フリック入力の方向規則をゲームパッドの左右の手に分離し、同時押しで1文字を入力する方式。

## 設計原則

1. **左手=子音、右手=母音**: ゲームパッドの物理的な左右分離を活用
2. **フリック順**: ボタン/LB/RB = 1番目（中央）、← = 2番目、↑ = 3番目、→ = 4番目、↓ = 5番目
3. **高頻度文字は単押し**: RB=あ段、RT=ん、LT+RT=っ
4. **後置シフト**: LT で拗音/小書き、右スティック↑で濁点/半濁点。入力後に変換する方式で同時押し数を削減
5. **Eager output + rollback**: 母音ボタン押下の瞬間に即出力し、閾値内の子音変更で巻き戻す

## 対応コントローラー

W3C Standard Gamepad Layout 準拠のコントローラー。ボタンは物理位置で判定するため、Xbox / PlayStation / Nintendo Switch Pro / 汎用 USB/Bluetooth コントローラーで動作する。

## ボタンマッピング

### 右手 = 母音（フェイスボタン + RB）

フリック入力の方向に対応:

| ボタン | Standard Gamepad Index | 母音 | フリック位置 |
|--------|----------------------|------|-------------|
| RB (R1) | 5 | あ (a) | 中央 |
| 左 (X/□) | 2 | い (i) | 2番目 |
| 上 (Y/△) | 3 | う (u) | 3番目 |
| 右 (B/○) | 1 | え (e) | 4番目 |
| 下 (A/×) | 0 | お (o) | 5番目 |

### 左手 = 子音行（D-pad + LB）

#### D-pad 単独（あ〜な行）

| D-pad | 行 | フリック位置 |
|-------|----|-------------|
| なし | あ行 | 中央 |
| ← | か行 | 2番目 |
| ↑ | さ行 | 3番目 |
| → | た行 | 4番目 |
| ↓ | な行 | 5番目 |

#### LB + D-pad（は〜わ行）

| 入力 | 行 | フリック位置 |
|------|----|-------------|
| LB | は行 | 中央 |
| LB + ← | ま行 | 2番目 |
| LB + ↑ | や行 | 3番目 |
| LB + → | ら行 | 4番目 |
| LB + ↓ | わ行 | 5番目 |

### かなテーブル（10行 × 5段）

```
            あ(RB)  い(←)  う(↑)  え(→)  お(↓)
なし:        あ      い      う      え      お
←(か行):    か      き      く      け      こ
↑(さ行):    さ      し      す      せ      そ
→(た行):    た      ち      つ      て      と
↓(な行):    な      に      ぬ      ね      の
LB(は行):   は      ひ      ふ      へ      ほ
LB+←(ま行): ま      み      む      め      も
LB+↑(や行): や      「      ゆ      」      よ
LB+→(ら行): ら      り      る      れ      ろ
LB+↓(わ行): わ      ゐ      ？      ゑ      を
```

※ や行の い/え 段にカギ括弧「」を配置
※ わ行の い/え 段に歴史的仮名 ゐ/ゑ を配置

## 特殊入力

### 単押し文字

| ボタン | 出力 | 判定方法 |
|--------|------|---------|
| RT 単押し | ん | RT リリース時、LT 関与なし |
| LT + RT 同時 | っ | LT ホールド中に RT が押された場合、LT リリースで発火 |

### 後置シフト（LT）

直前の composing 文字を変換する。1ボタンで3つの機能を兼ねる:

| 直前の文字 | LT 後置 | 結果 |
|-----------|---------|------|
| 母音（あ〜お） | → 小書き | あ→ぁ, い→ぃ, う→ぅ, え→ぇ, お→ぉ |
| あ/う/お段の子音 | → 拗音 | か→きゃ, く→きゅ, こ→きょ, ... |
| 濁音の子音 | → 濁音拗音 | が→ぎゃ, ぐ→ぎゅ, ご→ぎょ, ... |
| 半濁音の子音 | → 半濁音拗音 | ぱ→ぴゃ, ぷ→ぴゅ, ぽ→ぴょ |
| 上記以外 | → っ追加 | しゃ→しゃっ, ん→んっ, ... |

拗音対応行: か/さ/た/な/は/ま/ら行（あ/や/わ行は拗音なし → っ追加）

### 濁点/半濁点トグル（右スティック↑）

直前の文字の清音/濁音/半濁音をサイクル:

| 行 | トグル遷移 |
|----|-----------|
| か行 | か ⇔ が |
| さ行 | さ ⇔ ざ |
| た行 | た ⇔ だ |
| は行 | は → ば → ぱ → は（3段階） |
| う | う ⇔ ゔ |

## 右アナログスティック

| 方向 | 動作 | 判定 |
|------|------|------|
| ↑ | 濁点/半濁点トグル | ライジングエッジで1回発火 |
| → | 長音「ー」挿入 | 同上 |
| ← | BS（1文字削除） | 同上 |
| ↓ 1回 | 「、」挿入 | 同上 |
| ↓ 2回（400ms以内） | 「、」→「。」に差し替え | ダブルタップ判定 |
| 押し込み（RS） | キャンセル（composing 破棄） | リリースで発火 |

斜め入力防止: X/Y 軸の絶対値を比較し、大きい方のみ採用。閾値 0.5。

## 左アナログスティック（変換操作）

ネイティブアプリ（GIME）で実装済み。Web 版では未実装。

### 変換操作フロー

```
[composing] ひらがな入力中
    │
    │ 左スティック↓
    ▼
[selecting] 変換候補表示（カーソル付近に候補ウィンドウ）
    │
    │ 左スティック←→ で候補選択
    │ 左スティック↓ で次の候補群
    │ LS 押し込みで確定
    │ RS 押し込みでキャンセル → composing に戻る
    ▼
[confirmed] 確定テキスト
```

### マッピング

| 左スティック | composing 中 | selecting 中 |
|-------------|-------------|-------------|
| ← | 前の候補 | 前の候補 |
| → | 次の候補 | 次の候補 |
| ↑ | 前の候補 | 前の候補 |
| ↓ | 変換開始（selecting に遷移） | 次の候補 |

## 入力タイミング

### Eager Output + Rollback

母音ボタン押下の瞬間に即出力（eager output）し、同時打鍵ウィンドウ（300ms）内に子音が変化した場合は前の文字を差し替える（rollback）。

```
Frame 0: RB 押下 → 即 "あ" 出力
Frame 3: D-pad← 押下 (50ms後) → "あ" を "か" に差し替え（300ms以内）
Frame 5: LB 押下 (83ms後) → "か" を "ま" に差し替え（300ms以内）
...ボタンリリース → 確定
```

#### 巻き戻し防止

子音ボタンのリリース（ボタン数の減少）では巻き戻しを発火しない。ボタンの追加・切替時のみ発火する。

例: D-pad← + 下(A) で「こ」入力後、D-pad← を先に離しても「お」に戻らない。

### LT/RT 単押し判定

LT ホールド中に RT が押された場合は「っ」（LT+RT）。RT が押されなかった場合は LT 単押しで拗音後置シフト。LT リリース時に判定する。

### ネイティブ版（Swift、GIME アプリ）

| コンポーネント | 役割 |
|-------------|------|
| `GamepadResolver` | GCController → かな解決（gamepad-kana-table.ts の Swift 移植） |
| `InputManager` | かな漢字変換（既存、appendDirectKana 等で接続） |
| `GamepadVisualizerView` | SwiftUI ビジュアライザ |

アーキテクチャ:

```
キーボード:    pressesBegan → KeyEvent → KeyRouter → InputManager
ゲームパッド:  GCController → GamepadResolver → InputManager（直接注入）
ソフトキーボード: タッチイベント → FlickResolver → InputManager（同じパス）
```

## ビジュアライザ

ゲームパッド接続時にキーボードビジュアライザと差し替えて表示。

### 動的レイヤー切替

- **base レイヤー**: D-pad にあ〜な行名、フェイスボタンにあ行のかなを表示
- **LB レイヤー**: LB 押下中、D-pad には〜わ行名、フェイスボタンに対応行のかなを表示
- フェイスボタンの文字は左手の子音行に連動してリアルタイムに変化

### 右スティックビジュアライザ

ダイヤモンド配置で4方向の機能を表示。スティック傾きに連動してハイライト。中央に「取消」（RS 押し込み）。
